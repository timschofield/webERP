How to Fix It (Safely)
You just need to temporarily comment out these directives. It's a simple and reversible process.

1. 🛡️ Back Up the Configuration Files

Before editing anything, make a quick backup of the files. This lets you restore them easily.

Bash

sudo cp /etc/yum.conf /etc/yum.conf.bak
sudo cp /etc/yum.repos.d/almalinux.repo /etc/yum.repos.d/almalinux.repo.bak

2. 📝 Edit the Files

Open each file with a text editor like nano.

sudo nano /etc/yum.conf
Look for any of the lines mentioned in the warning (exclude, includepkgs, etc.) and add a # character to the beginning of that line. This "comments out" the line, making the system ignore it.

For example, if you find this line:
exclude=mariadb* php*

Change it to:
#exclude=mariadb* php*

Save the file (Ctrl+X, then Y, then Enter in nano).

Do the same for the other file:
sudo nano /etc/yum.repos.d/almalinux.repo

3. ✅ Re-run the Pre-Upgrade Check

Go back to the Plesk interface and run the pre-upgrade check again. The warning should now be gone.

4. 🎉 Proceed with the Upgrade

You can now safely continue with the Plesk upgrade process.

How to Fix the MariaDB Startup Failure
You need to create a systemd override file to grant the MariaDB service a higher file limit.

1. 🛡️ Create the Override Directory

First, create a directory for the MariaDB service override configuration. This is the standard way to modify service behavior on modern Linux systems like AlmaLinux.

Bash

sudo mkdir -p /etc/systemd/system/mariadb.service.d/

2. 📝 Create the Override Configuration File

copy file from github /etc/systemd/system/mariadb.service.d/override.conf into same path into server
cp /var/www/vhosts/kapal-laut.com/override.conf /etc/systemd/system/mariadb.service.d/override.conf

3. ✅ Reload Systemd and Restart MariaDB

For the system to recognize this new rule, you must reload the systemd daemon and then you can start MariaDB.

Bash

sudo systemctl daemon-reload
sudo systemctl restart mariadb
Your MariaDB service should now start successfully. The systemctl daemon-reload command tells the OS to read your new override.conf file, which now grants MariaDB the permission to open up to 65,536 files, matching the setting in your my.cnf.
