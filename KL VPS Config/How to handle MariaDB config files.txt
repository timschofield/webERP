/*************************************************************************
		How to handle MariaDB my.cnf file in server
*************************************************************************/


Changes made to MariaDB server variables through phpMyAdmin are typically applied as global variables and are not stored in a physical file. These changes are temporary and will be lost when the MariaDB server is restarted.

To make permanent changes, you need to edit the MariaDB configuration files, such as /etc/my.cnf or any .cnf files located in the /etc/my.cnf.d/ directory.

To restart the MariaDB service on your AlmaLinux server via the Plesk SSH terminal, use the following command:

systemctl restart mariadb

After the restart, you can verify the status of the service to ensure it is running correctly with:

systemctl status mariadb
